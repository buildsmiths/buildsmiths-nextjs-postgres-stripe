# Spec: AI Project Initialization

## Goal
Fully bootstrap this starter kit for local development. As an AI Agent, you should execute these steps automatically for the user when asked to "initialize", "set up", or "bootstrap" the project.

## Execution Steps

### 1. Environment Setup
- Check if `.env.local` exists. If not, copy `.env.example` to `.env.local`.
- Verify `NEXTAUTH_SECRET`. If it is empty or placeholder, generate a secure random 32-character string and inject it into `.env.local`.
- Ensure `NEXT_PUBLIC_SITE_URL` is explicitly set to `http://localhost:3000`.
- Ask the user to provide their local `DATABASE_URL` (Postgres) or use their current environment's default if available. Pause and wait for this if necessary before touching the database.

### 2. Dependencies
- Execute `npm install` to ensure all packages are strictly up to date.

### 3. Database Initialization
- Execute `npm run db:schema` to safely apply the idempotent schema to the Postgres database.
- If the schema applies successfully, run `npm run db:seed` to create a default local developer account.

### 4. Validation
- Run `npm run typecheck` to ensure there are no strict TypeScript compilation errors across the workspace.
- Run `npm run build` to confirm the Next.js App Router complies and builds cleanly.

## Verification & Hand-off
- Once all steps are completed successfully, instruct the user to start the development server by running `npm run dev`.
- Log the seeded credentials (if generated by `db:seed`) so the user can easily log in to the admin dashboard.
- Note: This project operates minimally without testing layers (Vitest/Jest). Focus purely on type safety and build steps.
